If you want your own CSS to override Bootstrap styles, the main trick is:

✅ Write your CSS AFTER Bootstrap

Example:

HTML
<head>
  <!-- Bootstrap CSS -->
  <link
    href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.7/dist/css/bootstrap.min.css"
    rel="stylesheet"
  >

  <!-- Your CSS -->
  <link rel="stylesheet" href="style.css">
</head>

Because style.css loads later, your styles get higher priority.

Example

Bootstrap Button

HTML
<button class="btn btn-primary custom-btn">
  Click Me
</button>

Your CSS

CSS
.custom-btn {
  background-color: black;
  border: none;
  color: white;
}

Now Bootstrap button becomes black instead of blue ✅

If Bootstrap Still Overrides Your CSS

Use:

1. More Specific Selectors

CSS
button.custom-btn {
  background-color: red;
}

OR

CSS
.container .custom-btn {
  background-color: red;
}

2. Use !important (Only when needed)

CSS
.custom-btn {
  background-color: red !important;
}

This forces your style to win.

Common Example

Override Navbar Color

CSS
.navbar {
  background-color: #111 !important;
}

Best Practice ⚡

Instead of editing Bootstrap classes directly:

❌ Avoid:

CSS
.btn {
  border-radius: 0;
}

✅ Better:

CSS
.my-btn {
  border-radius: 0;
}

Then:

HTML
<button class="btn btn-primary my-btn">

This keeps Bootstrap intact and your styles organized.

CSS Priority Order

Highest priority:

  1. Inline CSS

  2. !important

  3. Custom CSS loaded later

  4. Bootstrap CSS

Quick Real Example

HTML
<link rel="stylesheet" href="bootstrap.css">
<link rel="stylesheet" href="style.css">

<button class="btn btn-primary my-btn">
  Hello
</button>
CSS
.my-btn {
  background: purple;
  border-radius: 20px;
}

Result → Purple rounded Bootstrap button 🎉
